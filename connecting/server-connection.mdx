The moment LongFred has a WiFi connection and a valid IP address it starts looking for command stations on the same network. It does this using **mDNS service discovery**, the same zero-configuration protocol that lets smart speakers and printers announce themselves without any manual setup. You don't need to know the command station's IP address in advance — LongFred finds it for you. If discovery doesn't turn up anything, or you prefer to target a specific address, a manual entry screen is always available.

## Automatic mDNS discovery

After WiFi connects, LongFred sends mDNS queries for two service types in sequence:

- `_withrottle._tcp` — covers JMRI WiThrottle Server, DCC-EX, and any other WiThrottle-compatible software
- `_z21._udp` — covers Roco Z21 base stations

LongFred listens for responses for up to **10 seconds** in total. All responding stations are collected and shown on the **Server List** screen, each tagged with the protocol they speak:

| Tag | Protocol |
|-----|----------|
| **W** | WiThrottle |
| **Z** | Z21 (Roco) |

Up to **five** discovered servers are shown at a time.

<Note>
  If your SSID contains `DCCEX` or `DCC-EX`, LongFred skips the mDNS query and pre-fills a WiThrottle entry pointing at `192.168.4.1:2560` — the default address DCC-EX uses when running as its own access point. This saves the discovery wait time on a DCC-EX layout.
</Note>

## Auto-connect to the first WiThrottle server

LongFred is set to auto-connect: as soon as mDNS returns at least one WiThrottle server, LongFred connects to the first one automatically. You land straight on the throttle screen without needing to press anything.

<Tip>
  If your layout has exactly one WiThrottle command station and nothing else, auto-connect means the entire startup sequence — power on, WiFi join, discovery, connect — completes hands-free.
</Tip>

## Choosing a server from the list

If auto-connect is not appropriate for your session (for example, several servers are visible and you want a specific one), you can choose manually:

<Steps>
  <Step title="Browse the Server List">
    Use **Nav Up** or **Nav Down** to move the highlight through discovered servers. Each row shows the server's mDNS label, followed by its protocol tag (`W` or `Z`).
  </Step>
  <Step title="Connect">
    Press **OK** on the highlighted entry. LongFred opens a TCP connection (WiThrottle) or UDP session (Z21) to that server.
  </Step>
</Steps>

## Entering a server address manually

If mDNS finds nothing, or you want to reach a server that doesn't advertise itself, navigate past the discovered list to open the manual entry flow.

<Steps>
  <Step title="Open manual entry">
    From the Server List screen press **Nav Right** (shown in the hint as `> Manual`). LongFred shows the **Server Protocol** screen.
  </Step>
  <Step title="Choose the protocol">
    Use **Nav Up / Down** to highlight the protocol you need, then press **OK**:
    - `0 WiThrottle` — for JMRI, DCC-EX, and WiThrottle-compatible systems
    - `1 Z21` — for Roco Z21 base stations
  </Step>
  <Step title="Enter the IP address and port">
    The **Server Entry** screen appears with the default address pre-filled for the chosen protocol:

    | Protocol | Default address |
    |----------|----------------|
    | WiThrottle | `192.168.4.1:2560` |
    | Z21 | `192.168.0.111:21105` |

    Edit the digits using **F0–F9** (each key enters that digit). Use **Nav Up / Down** to increment or decrement the digit under the cursor. Press **OK** to confirm, or **Back** to cancel.
  </Step>
</Steps>

<Info>
  The IP and port are entered as a continuous string of digits — three digits per IP octet followed by five digits for the port (e.g. `19216800400102560` for `192.168.4.1:2560`). The display formats them with dots and a colon automatically so you can see the address clearly as you type.
</Info>

## Automatic reconnection

If the connection drops — because the command station restarted, the WiFi link flickered, or the session timed out — LongFred reconnects automatically using an **exponential back-off** strategy:

- First retry: **500 ms** after disconnect
- Each successive failure doubles the wait
- Maximum wait between retries: **5 000 ms**

This means momentary glitches resolve quickly while a completely unavailable server doesn't flood the network with retries. TCP keep-alive probes are sent every **5 seconds**, and the connection is declared dead if no response arrives within **8 seconds**.

<Warning>
  If LongFred can't re-establish a connection it will keep retrying indefinitely in the background. The throttle screen remains visible but commands will not reach the layout until the connection is restored.
</Warning>
