LongFred is a wireless DCC throttle controller that runs on an ESP32-C6 development board and connects to your model railway command station over WiFi. It gives you a physical, handheld control surface — rotary encoder, function buttons, navigation cluster, and an OLED display — without any cables between the throttle and the layout. Whether you run JMRI, DCC-EX, or a Roco Z21 system, LongFred speaks the right protocol and gets you driving trains within minutes of first boot.

## What LongFred Does

LongFred firmware scans for WiFi networks, authenticates with your chosen access point, discovers available command stations via mDNS, and then lets you acquire DCC locomotives by address and control them in real time. Up to six independent throttle slots are available, each capable of running a consist of up to ten locos. The OLED display shows speed, direction, active functions, and track power state at a glance.

## Supported Protocols and Command Stations

LongFred supports two industry-standard wireless DCC protocols:

| Protocol | Compatible Systems |
|---|---|
| **WiThrottle** | JMRI (any version with WiThrottle server), DCC-EX CommandStation-EX |
| **Z21** | Roco Z21 and compatible command stations |

When you reach the server selection screen, LongFred displays any servers it finds automatically via mDNS alongside the option to enter an IP address and port manually. The default WiThrottle address is `192.168.4.1:2560` (the DCC-EX access-point default) and the default Z21 address is `192.168.0.111:21105`.

## Compatible Hardware

LongFred targets the **ESP32-C6-DevKitC-1U** board paired with an SSD1306 OLED display and two MCP23017 I/O expanders for the button matrix. Refer to the [Hardware Overview](/hardware-overview) page for the complete pin map and wiring guide.
